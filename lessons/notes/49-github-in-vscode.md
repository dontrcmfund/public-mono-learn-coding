# GitHub in VS Code (first principles)

Goal: understand how VS Code connects to GitHub.

Why do we care?
- It lets you clone, sync, and push without extra steps
- It keeps your work backed up

First principles
- GitHub is the online copy of your repo
- VS Code can sign in to your GitHub account
- Once connected, syncing is simpler

What to do (slow and simple)
- Open the Account icon (bottom left)
- If asked, sign in to GitHub
- Notice VS Code can now access your repos

Checkpoint
- If you can say “VS Code can connect to GitHub,” you are ready to move on

Reflection
- Why might signing in feel intimidating?
- What would make it feel safer?
